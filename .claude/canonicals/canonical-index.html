# canonical-index.html Standard landing page structure for VLP ecosystem sites. --- ## Required Sections (in order) 1. **Hero** — headline, subheadline, primary CTA 2. **Value Proposition** — 3-4 benefit cards 3. **How It Works** — 3 steps 4. **Social Proof** — testimonials or stats 5. **Pricing Preview** — tier summary with link to /pricing 6. **FAQ** — 4-6 common questions 7. **Final CTA** — repeat primary call to action 8. **Footer** — links, legal, contact --- ## Rules - No external CSS frameworks — use the app's Tailwind config or CSS Modules - Mobile responsive (breakpoints per canonical-style.md) - All CTAs link to real routes within the app - No placeholder text — every word is final copy or marked `` - Brand color from the app's PlatformConfig (see canonical-stack.md for color values) - Jamie L Williams as contact where applicable - Must include proper `` tags for SEO (title, description, og:image) - Must include Cloudflare Web Analytics snippet where applicable - All images must have alt text - Links to external sites use `rel="noopener noreferrer"` and `target="_blank"` --- ## Monorepo Notes Landing pages live in each app's `app/page.tsx` (Next.js App Router) or equivalent. The shared `@vlp/member-ui` package provides structural components — landing pages are app-specific and do NOT use the shared member layout.
